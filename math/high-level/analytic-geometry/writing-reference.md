# 数学教材编写工作参考

> 基于《线性代数初步》的完整编写过程整理。供同类数学教材（课外扩展、高中高阶、大学入门）写作时参考。

---

## 一、核心理念

### 1.1 路线：从实例到归纳

定义和公式不是教材的起点，而是终点。每一章、每一节的正确顺序是：

```
具体数字例子（3~5 个）→ 观察模式 → 提出概念/公式 → 验证更多例子 → 练习巩固
```

**反例**（初期底稿的问题）：§3.5 先列出五行性质表，再给一句"举个例子"。学生看到表格时没有心理准备，不知道这些性质从哪来。

**修正后**：每条性质先给一组具体数字，让学生算出一个具体结果（如交换两行后 det 从 5 变 -5），再总结"由此可见，交换两行，行列式变号"。五条性质对应五组具体验算，表格作为最终的归纳总结放在最后。

### 1.2 原则：几何直觉承载代数结构

线性代数这类课程最大的教学风险是"过早抽象化"。全书的结构设计应该让每个代数概念都有一个对应的几何图像：

| 代数概念 | 几何图像 |
|----------|----------|
| 矩阵 | 平面上变换的"代码" |
| 矩阵乘法 | 两个变换的复合 |
| 行列式 | 面积/体积的放大倍率 |
| 行列式为 0 | 空间被压扁（降维） |
| 特征向量 | 变换中方向不变的"幸运方向" |
| 对角化 | 换到特征方向上看，变换变成纯伸缩 |
| 二次型的特征值 | 圆锥曲线旋转后的标准形系数 |

如果一个概念在你的教材中找不到对应的几何直觉，要么换一种讲法，要么接受它暂时只能作为"代数规则"被记住——但要明确告诉学生这一点。

## 二、章节结构模板

全书每一章应该遵循统一的结构骨架：

```
\chaptertitle{第X章\quad 标题}

[章首导语：3~5 行，说明本章要解决什么问题，与前面各章的关系]

\sectiontitle{X.1\quad 第一节标题}
  [正文 + 具体例子]
  \begin{exercise} [1~3 道即时练习] \end{exercise}

\sectiontitle{X.2\quad 第二节标题}
  ...

\sectiontitle{本章小结}
  [5~8 行，回顾本章核心概念、关键公式、与前后章的逻辑联系]

\sectiontitle{课后练习}
  \subsection*{A组\quad 基础巩固}  [6~10 题，纯计算熟练度]
  \subsection*{B组\quad 挑战提升}  [4~8 题，概念理解 + 跨节综合]
```

**检查清单**（每章验收时逐项核对）：
- [ ] 章首是否有导语？是否提到了前面的章节？
- [ ] 每节是否至少有 1 组即时练习？
- [ ] 是否有本章小结？
- [ ] A 组和 B 组是否齐全？
- [ ] 全章的 example/solution 环境中，是否有至少 3 个以上带具体数字的完整示例？

## 三、实例设计规范

### 3.1 实例的数量

每个新概念的引入至少需要 **3 个实例**，理想状态是 **4~5 个**。少于 3 个，学生无法辨认模式；多于 5 个，可能冗余——除非这些实例覆盖了不同类型的边界情况。

**正例**（ch03 §3.3 线性相关）：
- 实例 1：投影矩阵（列含零向量）
- 实例 2：列成 2 倍关系
- 实例 3：又是列成倍数（强化模式）
- 实例 4（对照）：列不成比例（正常情况，det≠0）
- 四个实例之后："归纳：上面四个实例揭示了同一个模式..."

### 3.2 实例的多样性

同一节内的多个实例应当覆盖概念的不同侧面，而不是简单地换数字。例如：

| 概念 | 实例应覆盖的侧面 |
|------|-----------------|
| 行列式为 0 | 投影（某列为零向量）、列成比例、某行全零、某两列相等 |
| 特征值正负号 | 两个正（椭圆）、一正一负（双曲线）、一零（抛物线）、两相等（圆） |
| 高斯消元 | 唯一解、无穷多解、无解——三种必须全部出现 |

### 3.3 实例中数字的"整洁度"

正文中的**引入性实例**应使用整洁数字（整数、简单分数），目的是让学生专注于概念本身而非计算细节。

课后的**巩固性练习**应有意识地混入"不整洁"数字——分数、根号、含参数——目的是让学生习惯真实场景。两种数字各有功能，不要互相替代。

## 四、练习设计规范

### 4.1 A 组与 B 组的分工

| | A 组（基础巩固） | B 组（挑战提升） |
|---|---|---|
| **目标** | 计算熟练度 | 概念理解 + 跨节综合 |
| **题型** | "算出来" | "解释为什么" / "举反例" / "证明" |
| **章节范围** | 当前章 | 可跨章 |
| **数字** | 适度"脏" | 可含参数、反推、开放构造 |

### 4.2 常见问题

**题量不足。** 初期底稿中 ch06 课后仅 10 题，ch05 仅 8 题。终稿各章 A+B 合计 12~18 题。

**例题即练习。** 初期底稿中 ch07 A 组 #2(c) 的方程 $x^2-6xy+9y^2=1$ 恰好是 §7.4 已经完整讲解过的例子——学生不需要思考，只需抄答案。验收时应逐一检查：每道 A/B 组题的矩阵/方程是否在正文 example 中出现过？如果出现过，替换为不同的数字或方程。

**跨章重复。** ch03 B 组有一道"已知 det 和 tr 求特征值"的题，但特征值的概念要到 ch05 才引入——这道题需要自带提示才能做，本质上是超前内容。验收时应检查每道题所需的概念是否已在前面章节中建立。

### 4.3 "脏"题的设计方法

所谓"脏"题，指答案不是整洁整数的题目。目的是训练学生在不凑整的情况下依然能正确执行算法。设计方法：

- 旋转角度不用 90°/180°，用 45°/135°/60°（产生 $\sqrt2/2$ 或 $\sqrt3/2$）
- 平移向量用 $(3,-2)$ 而非 $(2,0)$
- 矩阵元素用 $\frac13, \frac25$ 等分数
- 方程组系数使得解为分数而非整数
- 行列式中故意放一个 $\frac12$ 让学生处理通分
- 含参数 $k$ 的矩阵，问 $k$ 取何值时 det=0

## 五、常见初稿问题与修正方法

### 5.1 定义先于实例

**症状**：先用黑体字给出定义，再给一两个例子"验证"。

**修正**：先给 3~4 个例子，引导学生自行发现规律，最后用定义/公式作为归纳结论。

### 5.2 断言无验证

**症状**："A 是正交矩阵时保持长度不变"——只说结论，不给验证。

**修正**：挑一个具体的正交矩阵（如旋转 90° 的矩阵），选一个具体向量（如 $(3,4)$），算一遍变换前后的长度给学生看。只说"可以证明"而不证，在入门教材中等于没说。

### 5.3 节号与章号不同步

**症状**：ch05-eigenvalues.tex 的内容是第五章，但内部节号还标着 4.1~4.6。这是因为从旧版复制时忘记批量替换。

**修正**：每次调整章节编排后，用 grep 搜索所有 `\sectiontitle{` 和 `\chaptertitle{`，逐条与文件名和预期编号核对。

### 5.4 章间叙事断裂

**症状**：某章开头直接进入主题，没有一句话提到"前 X 章我们学了什么，这一章要解决什么新问题"。

**修正**：每章的第一段必须包含两个要素：(a) 回顾前面章节的关键收获；(b) 提出本章要回答的问题。例如 ch04 的章首导语："前三章我们掌握了变换的两个基本侧面：矩阵告诉我们每个点去了哪里，行列式告诉我们面积放大了几倍。现在问一个更深层的问题：在变换中，有没有某些方向是特殊的——它们不偏不转，只是被拉长或缩短？"

### 5.5 推广章与主线脱节

**症状**：ch04（从二维到 n 维）作为一个独立章插在行列式和特征值之间，内容极薄（仅 137 行），打断了叙事流。

**修正**：要么将推广内容融入各章（三维行列式放 ch03、n 维矩阵放 ch02、n 维特征值放 ch04），要么将它作为全书最后一章，系统性地对前面每一章做维度推广。终稿采用的是后者——第七章 §7.1~§7.6 分别对应前六章的 n 维推广。

## 六、审校流程参考

本教材经历了以下审校轮次，可作为同类项目的流程模板：

| 轮次 | 关注点 | 主要改动 |
|------|--------|----------|
| **第 1 轮** | 结构缺陷 | 节号错乱修复、第三章内容去重、ch04 位置决策 |
| **第 2 轮** | 内容深度 | ch03 线性相关扩充（实例→归纳）、ch02 高斯消元三种情况、ch05/06/07 章末小结补全 |
| **第 3 轮** | 练习质量 | 全等重复习题清理、A/B 组分工明确化、"脏"题补充 |
| **第 4 轮** | 叙事连贯 | 章首导语补全、节间过渡句、ch06 实例→归纳路线重写 |
| **第 5 轮** | 全局一致性 | 封面与使用说明、ch04 §4.4 推导补充、ch07 扩展 n 维仿射变换和二次型 |
| **第 6 轮** | 封装交付 | 习题量最终补充、答案逐条核对、目录/封面定稿 |

每轮的核心问题是循环递进的：**结构 → 内容 → 练习 → 叙事 → 扩展 → 封装**。

## 七、LaTeX 技术备忘

### 7.1 文件组织

推荐结构（本教材采用）：
```
analytic-geometry/
  classroom_materials.tex    % 主文档，\input 各章
  ch01.tex                   % 各章独立文件
  ch02.tex
  ...
  ch07-promotion.tex
  answers.tex                % 答案独立文件
../../teaching-style.sty     % 共享样式文件
```

主文档只做拼装：封面、使用说明、前言、`\input{ch01}`...`\input{answers}`。

### 7.2 中文编码注意事项

PowerShell 的 `(Get-Content) -replace | Set-Content` 命令在某些编码环境下会损坏中文 LaTeX 文件。**不要用它处理含中文的 .tex 文件**。中文替换应使用 Edit 工具逐处操作。

### 7.3 TikZ 插图

本教材共 7 幅 TikZ 图。关键经验：
- 使用 `mainblue` 和 `lightblue` 配色保持全书视觉一致
- 避免加载不必要的 TikZ 库（本教材仅用 `arrows.meta` 和 `calc`）
- 每幅图配一个居中的 `\node` 标题说明图意
- 复杂曲线（如倾斜椭圆）使用 `plot[smooth cycle, tension=0.8] coordinates {...}` 手工描点

### 7.4 自定义环境

本教材定义了以下 LaTeX 环境（见 `teaching-style.sty`）：
- `example` / `solution`：例题与解答，红色/蓝色标签
- `exercise`：练习题（enumerate 封装）
- `chaptertitle` / `sectiontitle`：章节标题（含 TOC 写入）
- `preface`：前言/说明类标题
- `maintitle`：封面大标题

这些环境确保了全书视觉风格的统一。同类项目建议也从定义专用环境开始。

## 八、写给下一本教材的作者

1. **先定骨架，再填血肉。** 在动手写任何一节正文之前，先列出全书的章-节标题树，确保每章有 5~7 节，逻辑链条上没有跳跃或缺口。

2. **每节至少 3 个实例。** 如果某节你写不出 3 个不同的、有代表性的实例，可能说明这个概念的引入方式需要重新设计——也许它不适合在这个学段讲，也许它需要和相邻节合并。

3. **练习不是正文的附庸。** 练习体系的设计应该和正文写作并行进行，而不是正文写完了再"配几道题"。A 组侧重熟练度，B 组侧重理解——两类题目在正文写作时就应该心中有数。

4. **让审校的人看"节号"和"重复"。** 这是两样作者最容易遗漏、审校者最容易发现的问题。每一轮审校都从这两项开始。

5. **封面留到最后。** 别在第一章还没写完时纠结封面设计。但交付前一定要做封面——它强迫你用 5 行字说清楚"这本书给谁看、讲什么、怎么讲"，这是对全书定位的最后一次检验。
